# Documentation

This directory contains supporting documentation for Smart Inquiry AI.

The root-level files describe the main project direction. The docs directory contains deeper planning notes that can evolve as the product becomes more concrete.

## Documents

- [architecture/ARCHITECTURE.md](architecture/ARCHITECTURE.md): High-level architecture, MVP deployment shape, and future expansion notes.
- [architecture/SYSTEM_ARCHITECTURE.md](architecture/SYSTEM_ARCHITECTURE.md): MVP system architecture and request flow.
- [database/DATABASE_SCHEMA.md](database/DATABASE_SCHEMA.md): MVP relational database schema.
- [database/ER_DIAGRAM.md](database/ER_DIAGRAM.md): Mermaid ER diagram and relationship rules.
- [api/API_CONTRACT.md](api/API_CONTRACT.md): MVP REST API contract.
- [api/ERROR_FORMAT.md](api/ERROR_FORMAT.md): Standard API error envelope and error handling rules.
- [domain/DOMAIN_MODEL.md](domain/DOMAIN_MODEL.md): Core domains, relationships, and business rules.
- [domain/GLOSSARY.md](domain/GLOSSARY.md): Shared terminology for the project.
- [decisions/DECISIONS.md](decisions/DECISIONS.md): Product and technical decision log.
- [MVP_SCOPE.md](MVP_SCOPE.md): MVP boundaries, required workflow, and non-goals.

## Wiki Structure

- `architecture/`: Architecture documentation.
- `api/`: API documentation.
- `database/`: Database documentation.
- `domain/`: Domain model and shared terminology.
- `decisions/`: Decision records.
- `learning/`: Learning notes.
- `meeting-notes/`: Meeting notes.
- `website/`: Website planning.
- `youtube/`: YouTube planning.
- `portfolio/`: Portfolio planning.

## Documentation Rules

- Keep product decisions visible and written down.
- Prefer small, clear documents over one oversized planning file.
- Update TODO.md when a planning item becomes actionable.
- Update CHANGELOG.md when project structure or major behavior changes.
- Avoid adding implementation details before the product scope is clear.
